# 定位功能说明

## 默认参考点逻辑

应用首次加载时的参考点优先级：

1. **已保存的自定义参考点**（最高优先级）
   - 如果用户之前通过"更换参照点"功能设置过参考点
   - 会从 localStorage 读取并使用

2. **当前位置（自动定位）**
   - 如果没有保存的参考点，会自动请求浏览器定位权限
   - 用户**允许定位**后：
     - 获取用户的地理坐标（WGS-84）
     - 转换为高德地图坐标系（GCJ-02）
     - 调用逆地理编码 API 获取具体地址信息
     - 显示实际地点名称（如"科大讯飞大厦"、"某某小区"等）
     - 同时显示完整地址（如"合肥市包河区某某路123号"）

3. **默认参考点：隆昊昊天园**（回退方案）
   - 用户**拒绝定位**时
   - 定位**超时**或**失败**时
   - 浏览器**不支持定位**时

## 预览环境 vs 实际使用

### 预览环境（当前）
- 浏览器出于安全考虑拒绝定位请求
- 显示默认的"隆昊昊天园"
- 这是**正常行为**，不影响实际功能

### 实际使用（生产环境）
用户首次访问页面时：

```
1. 页面加载
   ↓
2. 弹出浏览器定位权限请求
   ↓
3a. 用户点击"允许"              3b. 用户点击"拒绝"
   ↓                               ↓
4a. 定位成功 → 逆地理编码        4b. 显示"隆昊昊天园"
   ↓
5a. 显示具体地址
   例如：
   - 名称："合肥市政务中心"
   - 地址："合肥市蜀山区潜山路1469号"
```

## 技术实现

### 坐标转换
- 浏览器 Geolocation API 返回 **WGS-84** 坐标系
- 高德地图使用 **GCJ-02**（火星坐标系）
- 使用 `wgs84ToGcj02()` 进行坐标转换，避免数百米的偏移

### 逆地理编码
```javascript
reverseGeocode([longitude, latitude])
→ {
    name: "具体地点名称（优先POI）",
    address: "完整格式化地址"
  }
```

### 容错机制
- 定位超时：10秒
- 逆地理编码失败：显示"当前位置"
- 任何错误：自动回退到默认参考点

## 测试方法

### 在本地/生产环境测试定位功能：

1. 使用 **Chrome/Edge**（移动模拟器模式）
   - F12 → 设备模拟器
   - 传感器 → 位置 → 输入自定义坐标
   - 刷新页面测试

2. 使用真实移动设备
   - 清除浏览器缓存和 localStorage
   - 访问页面
   - 允许定位权限
   - 观察是否显示当前位置名称

3. **验证点**：
   - ✓ 首次访问弹出定位权限请求
   - ✓ 允许后显示逆地理编码获取的地点名称
   - ✓ 拒绝后显示"隆昊昊天园"
   - ✓ 再次访问不再请求定位（使用已保存的参考点）
